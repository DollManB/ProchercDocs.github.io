Player API
Инструменты для взаимодействия с игроком: сообщения, телепорт, позиция, ввод данных и многое другое. Объект Player доступен в скриптах, запущенных от имени игрока, либо получен через Server.
Методы
getName() → String
Получить никнейм игрока.
var name = Player.getName();
Player.sendMessage("Привет, " + name + "!");
sendMessage(text)
Отправить игроку сообщение в чат. Поддерживает §-коды цвета.
Player.sendMessage("§aЗелёный текст");
Player.sendMessage("§cКрасный текст");
teleport(x, y, z)
Телепортировать игрока на координаты.
Player.teleport(0, 64, 0);
getPos() → Vec3d
Получить текущую позицию игрока. Возвращает объект с полями .x, .y, .z.
var p = Player.getPos();
Player.sendMessage("X: " + p.x + " Y: " + p.y + " Z: " + p.z);
input(prompt, callback)
Запросить у игрока ввод текста через чат. Callback вызывается с введённой строкой.
Player.input("Введите ваше имя:", function(answer) {
  Player.sendMessage("Вы ввели: " + answer);
});
getServerPlayer() → ServerPlayerEntity
Получить нативный объект ServerPlayerEntity для низкоуровневых операций.
var sp = Player.getServerPlayer();

Советы

  • Проверяйте что Player не null, если скрипт запускается от сервера
  • Для отправки всем игрокам используйте Server.broadcast()
  • Чтобы получить PlayerAPI другого игрока — используйте Server.getPlayerByName()
Server API
Управление сервером: команды, список игроков, broadcast, тики.
Методы
executeCommand(command)
Выполнить команду от имени сервера (без слеша в начале).
Server.executeCommand("time set day");

// Выдать предмет игроку
var p = Server.getRandomPlayer();
Server.executeCommand("give " + p.getName() + " minecraft:diamond");
broadcast(message)
Отправить сообщение всем игрокам сервера.
Server.broadcast("§6[Сервер] Привет всем!");
getOnlinePlayer() → String[]
Получить список никнеймов всех онлайн-игроков.
var list = Server.getOnlinePlayer();
Server.broadcast("Онлайн: " + list.join(", "));
getPlayerCount() → int
Получить количество онлайн-игроков.
var count = Server.getPlayerCount();
getMaxPlayers() → int
Получить максимальное число слотов сервера.
var max = Server.getMaxPlayers();
getPlayerByName(nickname) → PlayerAPI
Найти игрока по никнейму. Возвращает PlayerAPI или null.
var p = Server.getPlayerByName("Steve");
if (p) p.sendMessage("Привет!");
getRandomPlayer() → PlayerAPI
Получить случайного онлайн-игрока.
var lucky = Server.getRandomPlayer();
if (lucky) lucky.sendMessage("§dТы выбран!");
getTicks() → long
Получить текущий тик сервера (20 тиков = 1 секунда).
var ticks = Server.getTicks();
Entity API updated
Создание и полное управление кастомными NPC: позиция при спавне через pos(), движение, взгляд, фракции, анимации. NPC используют GeckoLib-модели из ресурспака prochercengine_rp/.
Создание NPC
Entity.create(pos, name, modelPath, animPath, texturePath) → EntityHandle
Создать NPC в мире. Позиция задаётся через глобальную функцию pos(x, y, z). Пути к файлам — относительно папки prochercengine_rp/.
var npc = Entity.create(
  pos(0, -60, 0),                // позиция спавна
  "Охранник",                      // имя над головой
  "geo/plr_model.geo.json",        // GeckoLib .geo.json
  "animations/player.animation.json",
  "textures/stepan.png"
);
Движение
npc.move(pos) new
Отправить NPC идти к указанной позиции (стандартная скорость 1.0). NPC использует навигацию — обходит препятствия.
npc.move(pos(10, -60, 5));
npc.move(pos, speed) new
Идти к позиции с заданной скоростью (1.0 = норма, 1.5 = быстро, 2.0 = бегом).
npc.move(pos(10, -60, 5), 1.5);
Взгляд
npc.look(pos) new
Разово повернуть NPC в сторону позиции.
npc.look(pos(0, -60, 10));
npc.lookAlways(pos | PlayerAPI) new
Постоянно смотреть на указанную позицию или на игрока (каждый тик). Принимает pos(x,y,z) или объект PlayerAPI.
// Смотреть на точку вечно
npc.lookAlways(pos(0, -60, 0));

// Смотреть на конкретного игрока
var plr = Server.getRandomPlayer();
npc.lookAlways(plr);
npc.stopAlwaysLook() new
Остановить постоянный взгляд (NPC вернётся к обычному AI-поведению).
npc.stopAlwaysLook();
Фракции

Фракция определяет поведение NPC по отношению к игрокам. Сохраняется в NBT при перезагрузке мира.

"Neutral"
Мирный. Не нападает сам, но ответит ударом если его атаковать. Поведение по умолчанию.
"Aggressive"
Агрессивный. Видит игрока в радиусе 16 блоков → идёт к нему и атакует в ближнем бою.
"Peaceful"
Мирный абсолютно. Не атакует никогда, даже если его бьют.
var npc = Entity.create(pos(0, -60, 0), "Враг", ...);

npc.setFraction("Neutral");   // мирный, но даст сдачи
npc.setFraction("Aggressive"); // атакует при виде игрока
npc.setFraction("Peaceful");   // никогда не атакует

var f = npc.getFraction(); // → "NEUTRAL" / "AGGRESSIVE" / "PEACEFUL"
Остальные методы EntityHandle
setCustomName(name)
Установить имя над головой NPC (поддерживает §-цвета).
npc.setCustomName("§6Торговец");
setModel(modelPath)
Сменить GeckoLib-модель на лету.
npc.setModel("geo/new_model.geo.json");
setTexture(texturePath)
Сменить текстуру.
npc.setTexture("textures/new_skin.png");
setAnimation(animPath)
Сменить файл анимации.
npc.setAnimation("animations/idle.json");
say(text)
NPC пишет в общий чат от своего имени.
npc.say("Стой! Кто идёт?");
playAnimation(name, mode)
Запустить анимацию по имени. Режим: "loop", "once".
npc.playAnimation("attack", "once");
stopAnimation()
Остановить текущую кастомную анимацию.
npc.stopAnimation();
getPos() → Vec3d
Получить текущую позицию NPC.
var p = npc.getPos();
Server.broadcast("NPC на X:" + p.x);
getId() → int
Получить ID сущности в мире.
var id = npc.getId();
kill() / remove()
kill() — убить (с анимацией смерти). remove() — мгновенно удалить из мира без анимации.
npc.kill();  // с анимацией смерти
npc.remove(); // мгновенно
Автоматические анимации

NPC автоматически переключает анимации из файла анимаций — никакого кода не нужно. Названия в файле должны совпадать:

"idle"  → стоит на месте
"walk"  → идёт (velocity > 0.0001)
"run"   → бежит (isSprinting = true)

Советы

  • Файлы модели, анимаций и текстур кладите в prochercengine_rp/geo/, animations/, textures/
  • Фракция сохраняется в NBT — NPC "помнит" её после перезагрузки
  • lookAlways(plr) работает каждый тик — не нужен цикл
  • При Aggressive радиус видения игроков — 16 блоков
Sound API new
Воспроизведение звуков — у конкретного игрока, на позиции в мире, или сразу всем. Поддерживает кастомные звуки из ресурспака prochercengine_rp/sounds/ и стандартные звуки Minecraft (minecraft:...). Доступен как Sound в скриптах.
Воспроизведение
Конкретному игроку
Sound.playForPlayer(player, soundId)
Воспроизвести звук у конкретного игрока (только он его слышит). Громкость и высота — по умолчанию 1.0.
var plr = Server.getRandomPlayer();
Sound.playForPlayer(plr, "prochercengine:my_sound");

// Стандартный звук Minecraft:
Sound.playForPlayer(plr, "minecraft:entity.villager.yes");
Sound.playForPlayer(player, soundId, volume, pitch)
С кастомной громкостью и высотой тона. Volume: 0.0–2.0+. Pitch: 0.5 (низко) — 2.0 (высоко), 1.0 = норма.
Sound.playForPlayer(plr, "prochercengine:fanfare", 1.0, 1.5);
На позиции в мире
Sound.playAt(pos, soundId)
Воспроизвести звук в точке мира. Minecraft сам определяет кто услышит — зависит от расстояния до игроков.
Sound.playAt(pos(0, -60, 0), "prochercengine:explosion");
Sound.playAt(pos, soundId, volume, pitch)
С кастомными параметрами. Большая громкость увеличивает радиус слышимости.
Sound.playAt(pos(10, 64, -5), "minecraft:block.note_block.bell", 2.0, 0.8);
Всем игрокам
Sound.playAll(soundId)
Воспроизвести звук сразу всем онлайн-игрокам (каждый слышит "у себя").
Sound.playAll("prochercengine:lobby_music");
Sound.playAll(soundId, volume, pitch)
С кастомными параметрами.
Sound.playAll("prochercengine:countdown", 1.0, 2.0);
Остановка
Sound.stopForPlayer(player, soundId)
Остановить конкретный звук у игрока.
Sound.stopForPlayer(plr, "prochercengine:lobby_music");
Sound.stopAllForPlayer(player)
Остановить все звуки у игрока сразу.
Sound.stopAllForPlayer(plr);
Пример: счётчик-отсчёт
var count = 3;
while (count > 0) {
  Server.broadcast("§e" + count + "...");
  Sound.playAll("minecraft:block.note_block.pling", 1.0, 1.0);
  Time.wait(1.0);
  count--;
}
Server.broadcast("§aСтарт!");
Sound.playAll("minecraft:entity.player.levelup");

Советы

  • Кастомные звуки кладите в prochercengine_rp/sounds/ в формате .ogg
  • После добавления .ogg файлов запустите /prochercengine subscribe_sounds — sounds.json создастся автоматически
  • ID кастомного звука: prochercengine:имя_файла (без .ogg)
  • Все стандартные ID звуков Minecraft: minecraft.wiki
Hitbox API
Создание невидимых зон-триггеров. Поддерживает именованные хитбоксы (защита от дублирования при перезапуске скрипта), триггер входа, выхода и клика.
Создание
Hitbox.create(x, y, z, width, height) → HitBoxEntity
Создать анонимный хитбокс.
var zone = Hitbox.create(0, 64, 0, 3.0, 2.0);
Hitbox.create(name, x, y, z, width, height) → HitBoxEntity
Создать именованный хитбокс. При повторном вызове с тем же именем вернёт существующий (не создаст дубликат).
// Рекомендуемый способ — всегда давать имя
var zone = Hitbox.create("spawn_zone", 0, 64, 0, 4.0, 2.5);
onEnter — вход в зону

Срабатывает один раз при входе игрока. Повторно сработает только после выхода и нового входа.

var zone = Hitbox.create("my_zone", 0, -60, 0, 3.0, 2.0);

Hitbox.onEnter(zone.getEntity(), function(e) {
  e.player.sendMessage("§aВы вошли в зону!");
  // e.x, e.y, e.z — координаты игрока
});
onExit — выход из зоны
Hitbox.onExit(zone.getEntity(), function(e) {
  e.player.sendMessage("§cВы покинули зону");
});
onInteract — клик ПКМ
Hitbox.onInteract(zone.getEntity(), function(e) {
  e.player.sendMessage("§bВы нажали на хитбокс!");
});
Удаление
Hitbox.remove(zone.getEntity()); // удалить хитбокс

Важно

  • Всегда давайте хитбоксу имя через create("name", ...) — это предотвращает дублирование при перезапуске скрипта
  • Удаляйте хитбоксы при завершении игры/раунда через Hitbox.remove()
  • Рекомендуемый макс. размер: 10×10
Event API
Подписка на игровые события игрока: прыжок, присед, спринт, разрушение/установка блоков.
Методы
Event.onJump(player, callback)
Срабатывает когда игрок прыгает.
Event.onJump(Player.getServerPlayer(), function(e) {
  e.player.sendMessage("Прыжок!");
});
Event.onCrouch(player, callback)
Срабатывает при приседании/вставании.
Event.onCrouch(Player.getServerPlayer(), function(e) {
  e.player.sendMessage("Присел");
});
Event.onSprint(player, callback)
Срабатывает при начале/конце спринта. Объект события содержит isSprinting.
Event.onSprint(Player.getServerPlayer(), function(e) {
  if (e.isSprinting) {
    e.player.sendMessage("§6Бежишь!");
  }
});
Event.onBlockBreak(player, callback)
Срабатывает когда игрок ломает блок. Событие содержит player, x, y, z, block.
Event.onBlockBreak(Player.getServerPlayer(), function(e) {
  e.player.sendMessage("Сломал блок на " + e.x + " " + e.y + " " + e.z);
});
Event.onBlockPlace(player, callback)
Срабатывает при установке блока игроком.
Event.onBlockPlace(Player.getServerPlayer(), function(e) {
  e.player.sendMessage("Поставил блок!");
});
World API
Взаимодействие с игровым миром: установка блоков, взрывы.
Методы
World.setBlock(x, y, z, blockId)
Установить блок по координатам. ID блока в формате Minecraft namespace.
World.setBlock(0, 64, 0, "minecraft:diamond_block");
World.setBlock(0, 65, 0, "minecraft:air"); // убрать блок
World.spawnExplosion(x, y, z, power)
Создать взрыв в точке с заданной силой. TNT ≈ 4.0.
World.spawnExplosion(0, 64, 0, 3.0);
Item API
Создание кастомных предметов с GeckoLib-моделями, их выдача игрокам и спавн в мире.
Создание предмета
Item.create(name, modelPath, displayPath, animationPath) → ItemHandle
Зарегистрировать кастомный предмет с GeckoLib-моделью.
var sword = Item.create(
  "MagicSword",
  "geo/sword.geo.json",
  "display/sword.json",
  "animations/sword.json"
);
Методы ItemHandle
give(playerNameOrPlayerAPI)
Выдать предмет игроку по имени или через PlayerAPI.
sword.give("Steve");
sword.give(Player); // или через объект PlayerAPI
spawn(pos)
Заспавнить предмет в мире как дроп.
sword.spawn(pos(0, 65, 0));
getName() / getModelPath() / getAnimationPath()
Получить параметры предмета.
var name = sword.getName();
Particle API
Воспроизведение кастомных частиц из JSON-файлов в директории config/ProchercEngine/particles/.
Методы
Particle.spawn(name, x, y, z)
Воспроизвести частицу в координатах (видно всем).
Particle.spawn("explosion", 0, 64, 0);
Particle.spawn(name, x, y, z, player)
Воспроизвести частицу только для конкретного игрока.
Particle.spawn("sparkle", 0, 64, 0, Player.getServerPlayer());
Particle.spawn(name, entity)
Воспроизвести частицу на позиции сущности.
Particle.spawn("aura", npc.getEntity());

Формат файла частицы

Файлы частиц хранятся в config/ProchercEngine/particles/ с расширением .particle.json.

Time API
Управление игровым временем: заморозка, установка, ожидание в скриптах.
Методы
Time.wait(seconds)
Приостановить выполнение скрипта на N секунд.
Time.wait(3.0); // ждать 3 секунды
Time.setTime(ticks)
Установить время суток (0–24000).
Time.setTime(6000); // полдень
Time.setTime(18000); // полночь
Time.freezeTime()
Заморозить течение времени.
Time.freezeTime();
Time.freezeResume()
Возобновить течение времени.
Time.freezeResume();
Тики
0 = Рассвет (6:00)
6000 = Полдень (12:00)
12000 = Закат (18:00)
18000 = Полночь (0:00)
24000 = Конец цикла
Ресурс-пак new
Структура папки prochercengine_rp/ — кастомные модели, анимации, текстуры и звуки для NPC и предметов. Создаётся командой /prochercengine create_resource.
Структура папок
prochercengine_rp/
├── pack.mcmeta         // создаётся автоматически
├── sounds.json        // регистрация звуков (auto через subscribe_sounds)
├── geo/               // GeckoLib .geo.json модели
│   └── plr_model.geo.json
├── animations/        // .animation.json файлы
│   └── player.animation.json
├── textures/          // .png текстуры
│   └── stepan.png
└── sounds/            // .ogg звуки
    ├── fanfare.ogg
    └── ambient/
        └── fire.ogg
Добавление звуков — пошагово
Шаг 1 — Создать структуру
Если ресурспак ещё не создан, запустите команду:
/prochercengine create_resource
Шаг 2 — Добавить .ogg файлы
Положите ваши .ogg файлы в папку prochercengine_rp/sounds/. Можно создавать подпапки.
prochercengine_rp/sounds/fanfare.ogg
prochercengine_rp/sounds/ambient/rain.ogg
Шаг 3 — Зарегистрировать звуки
Запустите команду — она найдёт все .ogg и сгенерирует sounds.json автоматически:
/prochercengine subscribe_sounds

// Результат в sounds.json:
{
  "fanfare": { "sounds": [{ "name": "prochercengine:fanfare" }] },
  "ambient/rain": { "sounds": [{ "name": "prochercengine:ambient/rain" }] }
}
Шаг 4 — Использовать в скрипте
После перезагрузки ресурспака звуки доступны в Sound API:
Sound.playAll("prochercengine:fanfare");
Sound.playAll("prochercengine:ambient/rain");
Модели GeckoLib

Модели создаются в Blockbench с форматом GeckoLib Entity. Экспортируйте как .geo.json в папку geo/, анимации как .animation.json в animations/.

// Используем в Entity.create()
var npc = Entity.create(
  pos(0, -60, 0),
  "Гвардеец",
  "geo/guard.geo.json",       // из geo/
  "animations/guard.animation.json", // из animations/
  "textures/guard.png"        // из textures/
);

Советы

  • Ресурспак живёт рядом с сервером (там же где server.jar), а не в config/
  • Звуки обязательно должны быть в формате .ogg — другие форматы не поддерживаются Minecraft
  • После изменения sounds.json игрокам нужно перезагрузить ресурспак (F3+T)
  • Анимации должны содержать состояния idle, walk, run для автопереключения
Команды движка new
Серверные команды для управления скриптами и ресурспаком. Все команды начинаются с /prochercengine.
Скрипты
/prochercengine run-script <name>
Запустить скрипт по имени (без расширения .pec). Скрипты хранятся в config/ProchercEngine/scripts/.
/prochercengine run-script my_minigame
// запустит config/ProchercEngine/scripts/my_minigame.pec
Ресурс-пак
/prochercengine create_resource
Создаёт папку prochercengine_rp/ со всей структурой и базовыми файлами. Безопасно — существующие файлы не перезаписывает.
/prochercengine create_resource

// Создаёт:
// prochercengine_rp/geo/
// prochercengine_rp/animations/
// prochercengine_rp/textures/
// prochercengine_rp/sounds/
// prochercengine_rp/pack.mcmeta
// prochercengine_rp/sounds.json
/prochercengine subscribe_sounds
Сканирует папку prochercengine_rp/sounds/ рекурсивно, находит все .ogg файлы и автоматически генерирует / перезаписывает sounds.json. Запускать каждый раз после добавления новых звуков.
/prochercengine subscribe_sounds

// Пример вывода:
// [Engine] sounds.json обновлён! 3 звуков:
// • prochercengine:fanfare
// • prochercengine:countdown
// • prochercengine:ambient/rain
Структура скрипта (.pec)

Скрипты пишутся на JavaScript-подобном языке ProchercScript. Файлы с расширением .pec. Комментарий //autorun в первой строке запустит скрипт автоматически при старте сервера.

// Пример: простой NPC-охранник

var npc = Entity.create(
  pos(0, -60, 0),
  "Охранник",
  "geo/guard.geo.json",
  "animations/guard.animation.json",
  "textures/guard.png"
);

npc.setFraction("Neutral");

var zone = Hitbox.create("guard_zone", 0, -60, 0, 5.0, 2.5);

Hitbox.onEnter(zone.getEntity(), function(e) {
  npc.lookAlways(e.player);
  npc.say("Стой! Кто идёт?");
  Sound.playForPlayer(e.player, "minecraft:entity.villager.ambient");
});

Hitbox.onExit(zone.getEntity(), function(e) {
  npc.stopAlwaysLook();
});

Советы

  • Скрипты запускаются в отдельных потоках — несколько скриптов могут работать одновременно
  • Используйте Time.wait() внутри скриптов для задержек — не Thread.sleep()
  • После изменения звуков всегда запускайте subscribe_sounds, потом попросите игроков нажать F3+T
