Player API
Player API предоставляет полный набор инструментов для работы с игроками в игре. Этот API позволяет управлять всеми аспектами игрока: от базовой информации до сложных взаимодействий.
Лучшие практики

Обработка ошибок

Проверяйте возвращаемые значения на null/undefined

Ограничения
  • Некоторые методы могут не работать с не-игровыми сущностями
  • Модификации инвентаря могут конфликтовать с другими модами
  • Физические изменения могут не синхронизироваться в реальном времени в мультиплеере
Методы API
getName()
Получить имя игрока
let name = Player.getName();
// пример использования
let name = Player.getName();
Player.sendMessage(name);
sendMessage(text)
Игрок может разговаривать, выводить сообщение в чат
Player.sendMessage("text{STRING}");

// с переменной:
const text = "hi";
Player.sendMessage(text);
getServerPlayer()
Получить ServerPlayerEntity - компонент игрока зависящий от net.minecraft.server.network

Полную информацию можно посмотреть тут

let player = Player.getServerPlayer(); // возвращает ServerPlayerEntity в player
📦 HitboxAPI
API который поможет вам в разработке своим хитбоксом, он выполняет две функции: InteractTrigger и enteredTrigger
interactTrigger

Работает по нажатию ПКМ по хитбоксу

// Пример использования interactTrigger
var myHitbox = Hitbox.create(100, 64, 100, 2.0, 3.0);
myHitbox.onInteract(myHitbox.getEntity(), function(Player) {
  // действие
});
enteredTrigger

Работает когда игрок зашел в триггер

// Пример использования enteredTrigger
var myHitbox = Hitbox.create(100, 64, 100, 2.0, 3.0);
myHitbox.onEnter(myHitbox.getEntity(), function(Player) {
  // действие
});
Удаление хитбокса

А так же можно убирать его

// Пример удаления хитбокса
var myHitbox = Hitbox.create(100, 64, 100, 2.0, 3.0);
myHitbox.remove();

Рекомендации по использованию

  • Используйте interactTrigger для интерактивных объектов (двери, кнопки, сундуки)
  • Используйте enteredTrigger для зон (лава, вода, телепорты)
  • Не забывайте удалять хитбоксы после окончания раунда и тд, чтобы ничего не багалось
  • Координаты (x, y, z) задают центр хитбокса, а (width, height) - его размеры

Ограничения

  • Минимальный размер хитбокса: 1x1
  • Максимальный размер хитбокса по рекомендации: 10x10(иначе он выйдет слишком большим)
Server API
Server API предоставляет мощные инструменты для управления сервером и глобальными игровыми процессами. Этот API позволяет контролировать все аспекты серверной части игры, от управления игроками до выполнения команд и модификации игрового мира.
Методы API
executeCommand(command)
Этот метод позволяет от имени сервера выполнить команду
Необязательно писать через [ / ]
Server.executeCommand("example command");

// пример:
Server.executeCommand("time set day");

// еще пример:
let plr = Server.getRandomPlayer();
let name = plr.getName();
Server.executeCommand("give " + name + " minecraft:stick");
getOnlinePlayer()
Получить список игроков (имена)
// пример:
let list = Server.getOnlinePlayer();
getOnlinePlayer()
Получить список игроков (имена)
// пример:
let list = Server.getOnlinePlayer();
getPlayerCount()
Получить число игроков на сервере
// пример:
let count = Server.getPlayerCount();
getPlayerByName(nickname)
Получить игрока по имени
// пример:
let player = Server.getPlayerByName("Nickname"); // передает PlayerAPI
// после этого теперь player у нас является зависимым от PlayerAPI
getRandomPlayer()
Получить рандомного игрока
// пример:
let player = Server.getRandomPlayer(); // теперь player у нас становиться зависимым от PlayerAPI
broadcast(message)
Отправляет сообщение в чат от имени сервера
// пример:
Server.broadcast("привет мир!");
Полный пример использования
// Пример: Выдать рандомному игроку алмаз
function giveRandomPlayerDiamond() {
  let randomPlayer = Server.getRandomPlayer();
  if (randomPlayer) {
    let playerName = randomPlayer.getName();
    Server.executeCommand("give " + playerName + " minecraft:diamond");
    Server.broadcast("Игрок " + playerName + " получил алмаз!");
  }
}
Time API
Time API предоставляет инструменты для управления временем в игре. Этот API позволяет контролировать игровое время, устанавливать задержки и управлять его потоком.
Методы API
wait(seconds)
Ожидание на указанное количество секунд
// Ждать 1 секунду
Time.wait(1.0);

// Ждать 0.5 секунды
Time.wait(0.5);

// Ждать 3 секунды
Time.wait(3.0);
freezeTime()
Замораживает игровое время
// Заморозить время
Time.freezeTime();
freezeResume()
Восстанавливает нормальное течение времени
// Разморозить время
Time.freezeResume();
setTime(ticks)
Устанавливает время в игре (0-24000)
// Установить время на рассвет (0)
Time.setTime(0);

// Установить время на полдень (6000)
Time.setTime(6000);

// Установить время на закат (12000)
Time.setTime(12000);

// Установить время на полночь (18000)
Time.setTime(18000);
Примеры использования
// Пример: Создать закат через 5 секунд
function createSunset() {
  Time.wait(5.0);
  Time.setTime(12000);
  Server.broadcast("Наступил закат!");
}

// Пример: Заморозить время на 10 секунд
function freezeTimeForSeconds(seconds) {
  Time.freezeTime();
  Time.wait(seconds);
  Time.freezeResume();
  Server.broadcast("Время снова идет!");
}
Информация о времени
Игровое время измеряется в тиках (0-24000)
// Временные метки:
0    = Рассвет (6:00 утра)
6000    = Полдень (12:00 дня)
12000   = Закат (6:00 вечера)
18000   = Полночь (12:00 ночи)
24000   = Полный цикл (возвращается к 0)
Ограничения
  • Время должно быть в диапазоне от 0 до 24000
  • Метод wait() блокирует выполнение скрипта на указанное время
  • freezeTime() и freezeResume() работают только на сервере
  • Изменения времени могут не синхронизироваться мгновенно у всех игроков
